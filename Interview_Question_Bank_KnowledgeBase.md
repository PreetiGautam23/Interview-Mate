# Interview Question Bank & HR Guidelines (Knowledge Base)

## Technical Round — Software/IT Roles (Freshers)

1. What is the difference between a stack and a queue? Where would you use each?
2. Explain the concept of Object-Oriented Programming with a real example.
3. What is the difference between SQL and NoSQL databases?
4. Explain how a hash table works and its time complexity for lookup.
5. What is the difference between REST and GraphQL APIs?
6. Walk me through how you would design a URL shortener.
7. What is edge computing, and how does it differ from cloud computing?
8. Explain the CAP theorem in distributed systems.

## Behavioral Round — Common Questions

1. Tell me about yourself.
2. Describe a challenging project and how you handled it.
3. Tell me about a time you disagreed with a team member. How did you resolve it?
4. Where do you see yourself in 5 years?
5. Why should we hire you over other candidates?
6. Describe a time you failed at something. What did you learn?
7. How do you handle tight deadlines and pressure?

## HR Round Guidelines

- Be concise: aim for 60–90 second answers for behavioral questions.
- Use the STAR method (Situation, Task, Action, Result) for behavioral answers.
- Research the company beforehand — mention specific products, values, or recent news.
- Always prepare 2–3 thoughtful questions to ask the interviewer at the end.
- Dress appropriately and test your video/audio setup in advance for virtual interviews.

## Role-Specific Focus Areas

| Role | Key Focus Areas |
|------|-----------------|
| Software Developer | DSA, OOP, System Design basics, one strong project deep-dive |
| Data/ML Intern | Statistics, Python, one end-to-end ML project explanation |
| IoT/Edge Computing | Sensor networks, embedded systems, real-time data processing |
| Web Developer | Frontend/backend frameworks, REST APIs, deployment basics |

## Model Answer Example

**Q: Tell me about yourself.**
A strong answer covers: (1) a one-line professional identity, (2) 2–3 relevant
achievements or projects tied to the role, (3) why this role/company excites you.
Keep it under 90 seconds and avoid reciting the entire resume.

## Common Mistakes to Avoid

- Giving one-word or overly generic answers ("I'm a hard worker").
- Speaking negatively about a previous college, team, or professor.
- Not asking any questions back to the interviewer.
- Rambling without a clear structure (use STAR for behavioral questions).
